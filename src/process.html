Arbetsprocess

Innan kodning

Jag började med att ladda hem "Homebrew" till datorn. Detta för att sedan kunna köra GitHub genom Visual Studio Code. För att ladda hem "Homebrew" använde jag terminalen på datorn och klistrade in det som stod på deras hemsida.

Efter nedladdningen av "Homebrew" kunde jag köra GitHub genom Source Control-fliken i Visual Studio Code.

När detta var fixat började jag med skapandet av demowebbplatsen. Jag började med att skapa en mapp i kodeditorn och därefter skrev jag in "npm init. Efter detta skrev jag in "npm install parcel --save-dev" för att installera Parcel.

När detta var gjort ändrade jag lite i package.json-filen. Jag lade till "source": "src/index.html" och även "scripts" för start och build. Jag lade även till författare till mitt namn.

Skapande av webbplats

Efter detta skapade jag en .gitignore-fil för att exkludera node_modules och andra filer i versionshanteringen. Här skapade jag även en "src"-fil där HTML, CSS och JavaScript-filerna lades.

Skapande av meny började jag med sedan. Jag gjorde en enkel meny med tre menyval och även en responsiv meny med hjälp av JavaScript. Efter detta gjorde jag min första commit till GitHub, men innan det skapade jag en ny branch med namn "development", som skulle stå för utveckling.

Efter menyn var klar, skapade jag startsidan där jag beskrev uppgiften.

När startsidan var klar gjorde jag en till commit till GitHub och därefter började jag med bild-sidan. Jag tog fem bilder från min telefon och lade in dessa i en mapp under "src"-mappen som fick heta "pics."

Bild-sidan

När bilderna var vald och inlagd i projektet, skapade jag picture-element för att sedan kunna koda för att automatiskt optimera bilderna. Det var kul att se bilderna ändras direkt när man ändrar koden och även vilken skillnad det blev på filstorlekar.
Bilderna är PNG-bilder från början. Jag skrev kod för att ändra bilderna till avif i första taget med en bredd på 500. Om inte avif - webp och om inte någon av dessa format fungerar körde jag PNG-bilden som fallback i samma bredd. Bilderna lades in på hemsidan med två bilder per rad och en ensam bild längst ned. När skärmbredden minskas läggs bilderna vertikalt automatiskt.

När bilderna var implementerade till koden gjorde jag ytterligare en commit till GitHub och därefter påbörjade jag denna sida.

Det sista

Det sista jag implementerade var "browserslist" i package.json-filen. Jag skrev denna med "> 0.5%, last 2 versions, not dead". Sedan gjorde jag en "npm run build" för att komprimera filerna.
Därefter var det bara att göra commit och merge till main-branchen innan publicering.
